Class CipherOutputStream

  • java.lang.Object
    • java.io.OutputStream
      • net.java.truecommons.io.DecoratingOutputStream
        • net.java.truevfs.comp.zip.crypto.CipherOutputStream
  • All Implemented Interfaces:
    Closeable, Flushable, AutoCloseable

    @NotThreadSafe
public final class CipherOutputStream
extends net.java.truecommons.io.DecoratingOutputStream
    Similar to javax.crypto.CipherOutputStream with some exceptions:
    • This implementation is based on Bouncy Castle's lightweight crypto API and uses a BufferedBlockCipher for ciphering.
    • The flush() method just flushes the underlying output stream and has no effect on the cipher.
    • For compensation, the finish() method finishes the output (probably producing padding bytes) without closing the output. This can get used in clients to produce a trailer with additional information about the ciphered data (e.g. a MAC).

        public void finish()
            throws IOException
        Finishes and voids this cipher output stream. Calling this method causes all remaining buffered bytes to get written and padded if necessary. Afterwards, this stream will behave as if it had been closed, although the decorated stream may still be open.
