Compatibility to Previous Releases

In general, an updated minor version number indicates that the public API has been kept backward compatible to the previous release. Thus, client applications should not need to get recompiled. However, there is one exception: A feature may be changed if and only if all of the following conditions apply:

  1. The feature's design is broken.
  2. The feature is assumed to be rarely used by client applications or the implications of not changing it are considered to be unacceptable.
  3. The change is documented in the update list of the Release Notes.
  4. The change is marked as "Changed" in the update list.
  5. The change is justified in the update list.
  6. A workaround is explained in the update list.

If your client application is affected by a change and the documented workaround is unacceptable for any reason, please submit this issue to the project's Issue Tracking System.
