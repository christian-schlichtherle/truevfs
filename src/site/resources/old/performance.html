TrueZIP outperforms 7-Zip

The following example demonstrates the performance of unzipping the source archive from JDK 1.5.0_08 for the Windows platform to a directory named "src" in the current directory.

The performance snapshot has been taken within a bash shell (from CygWin) with TrueZIP 6.2.1 and 7-Zip 4.42 on Windows XP (all Service Packs installed up to Sep. 7th, 2006) on P4 3.4 GHz/Hyperthreading. Results are stable on multiple invocations.

Relevant information has been set in bold font. 

$ rm -rf src; time /cygdrive/c/Programme/7-Zip/7z x -osrc src.zip >NUL:

real    0m35.032s
user    0m0.015s
sys     0m0.000s

$ rm -rf src; time /cygdrive/c/Programme/Java/jdk1.5.0_08/bin/java -jar truezip-6.jar cp -unzip src.zip src

real 0m31.641s
user 0m0.015s
sys 0m0.015s

$ rm -rf src; time /cygdrive/c/Programme/Java/jdk1.5.0_08/bin/java -noverify -jar truezip-6.jar cp -unzip src.zip src

real 0m27.984s
user 0m0.015s
sys 0m0.015s

$ rm -rf src; time /cygdrive/c/Programme/Java/jdk1.5.0_08/bin/java -noverify -server -jar truezip-6.jar cp -unzip src.zip src

real 0m28.688s
user 0m0.015s
sys 0m0.015s

As you can see, within this example, TrueZIP performs always slightly better than 7-Zip. Results may vary depending on OS platform, JDK implementation and input archive.
