TrueZIP is a framework for virtual file systems and a library for accessing archive files as if they were just plain old directories.

As a framework, TrueZIP provides the interfaces and classes to write file system drivers which plug-in to its federated file system space.

As a library, TrueZIP's File* API provides convenient multi-threaded read/write access to archive files as if they were just plain old directories in a file system path.

Use Cases

Core Concept and Definitions

The core concept of TrueZIP is the federation of virtual file systems into a file system space which can be transparently accessed via uniform calls to a convenient and comprehensive API. To understand what this means, let's check the following definitions first:

File System
An atomic, persistent container for files and directories in order to store and structure arbitrary data. Normally, the operating system provides an application with the means to access a set of one or more file systems like NTFS or ext4.
Virtual File System
A component which provides the same services like a file system, but is synthesized at run time from one or more components which are not file systems by themselves.
Federated File System
A file system with a root directory which is a member within another (parent) file system, which may be federated, too.
File System Space
The union of all file systems.

Note that the attributes virtual and federated in their strict sense are complementary, so a file system could be one or the other or both at the same time.

As an example for a file system which is virtual, but not federated, consider the HTTP(S) protocols: With their methods PUT, GET, POST and DELETE and the URI adressing scheme, the HTTP(S) protocols provide a subset of typical file system services.

As an example for a file system which is federated, but not virtual, consider a native Unix file system which is mounted at a directory path within the operating system's file system space, like e.g. {@code /mnt/cdrom}.

As an example for a file system which is both virtual and federated, consider any archive file which is accessed via TrueZIP's API: It's virtual because it's adapted to typical file system services by TrueZIP and its federated because its a file member within another file system - which is a native file system of the operating system in most cases.

Within the context of TrueZIP, the terms federated virtual file system, virtual federated file system and federated file system are used as synonyms. This simplification should do no harm because as a Java application, TrueZIP provides only federated file systems which are virtual file systems at the same time, too.

Key Features

Easy To Use
The File* API provides convenient read/write access to archive files as if they were just plain old directories in a file system path. Using the classes {@link de.schlichtherle.truezip.file.TFile}, {@code de.schlichtherle.truezip.file.TFileInputStream}, {@code de.schlichtherle.truezip.file.TFileOutputStream} et al, you can instantly access ZIP files, TAR files and their derivatives or your own custom archive file format.
Fast Bulk I/O
The File* API provides convenient and powerful methods for fast bulk I/O operations like recursive copying, moving or deleting of directory trees and files. E.g. using the multi-threaded method {@link de.schlichtherle.truezip.file.TFile#cat(java.io.InputStream, java.io.OutputStream)}, you'll never have to write a naive read-then-write copy loop again.
File System Federation
TrueZIP supports virtual file system federation. This enables an application to access archive files within any other virtual file systems, e.g. you could read the contents of a ZIP file which is accessed via HTTP(S) instead of reading it from a file. Archive files may also be nested, i.e. contained in another archive file. This is a generalization of the concept implemented by the class {@link java.net.JarURLConnection}.
Extensible Driver Architecture
TrueZIP supports any virtual file system type via its pluggable {@link de.schlichtherle.truezip.fs.FsDriver file system driver} architecture. TrueZIP ships with file system drivers for HTTP(S), ZIP, TAR and their derivatives (JAR, TGZ, TBZ2 etc).
Thread-Safe
Multiple threads can read and write the same archive file at the same time.
Robust
TrueZIP code uses assertions, JUnit tests and static code analysis tools like FindBugs and JDepend in order to achieve its high reliability standards.
Encrypted Archive File Format
TrueZIP includes a reference implementation for it's custom Random Access Encryption Specification (RAES). RAES is a generic wrapper file format which supports SHA-256 authentication and AES-256 encryption in CTR block mode for random read access. When combined with the ZIP File Format, this results in the encrypted archive file format called ZIP.RAES or TZP for short. TrueZIP ships with a file system driver to enable transparent read/write access to TZP files.
