5   package net.java.truevfs.access;
6   
7   import net.java.truevfs.kernel.spec.FsSyncException;
8   import net.java.truevfs.kernel.spec.FsSyncWarningException;
9   
10  /**
11   * A template class which aids in establishing the typical
12   * <i>setup-work-sync</i> life cycle of a TrueVFS application.
13   * The established life cycle consists of the {@link #setup} phase and one or
14   * more iterations of the {@link #work} and finally {@link #sync} phases.
15   * <p>
16   * Subclass or copy-and-edit the source code of this template class to meet
17   * your requirements.
18   * <p>
19   * 
20   * @param   <E> the {@link Exception} class to throw by {@link #work} and
21   *          {@link #run}.
22   * @author Christian Schlichtherle
23   */
24  public abstract class TApplication<E extends Exception> {
25  
26      /**
27       * Runs the setup-work-sync life cycle.
28       * At first, this method calls {@link #setup}.
29       * Then, a loop is entered which calls {@link #work} and (in a finally
30       * block) {@link #sync}.
31       * If {@link #work} returns a negative integer, then the loop is
32       * repeated.
33       * Otherwise, the return value is used as the
34       * {@link System#exit(int) exit status} of the VM.
35       * <p>
36       * Note that calling {@link #sync} in a finally-block ensures that all
37       * unsynchronized changes to the contents of all federated file systems
38       * (i.e. archive files) get committed to their respective parent file
39       * system, even if {@link #work} throws an exception.
40       * 
41       * @param  args an array of arguments for this application.
42       * @return The exit status of this application.
43       * @throws E At the discretion of the {@link #work} method.
44       * @throws FsSyncException At the discretion of the {@link #sync} method.
45       */
46      protected final int run(String[] args) throws E, FsSyncException {
47          setup();
48          int status;
49          do {
50              try {
51                  status = work(args);
52              } finally {
53                  sync();
54              }
55          } while (0 > status);
56          return status;
57      }
58  
59      /**
60       * Runs the setup phase.
61       * <p>
62       * This method is called by {@link #run} at the start of the life cycle.
63       * Its task is to configure the default behavior of the TrueVFS Access File* API
64       * in order to answer the following questions:
65       * <ul>
66       * <li>Which file extensions shall get detected to indicate archive
67       *     files and hence as virtual directories?
68       * <li>Which kind of temporary buffers shall get used?
69       * <li>Shall missing archive files and directory entries get automatically
70       *     created whenever required?
71       * </ul>
72       * <p>
73       * Note that the method body of the implementation in the class
74       * {@link TApplication} is empty!
75       * This means that the initial setup is not changed.
76       * The initial setup is determined by loading service classes from the
77       * classpath and applying reasonable defaults if nothing is found.
78       * <p>
79       * As an alternative to overriding this method, the setup may get changed
80       * in a sub class constructor.
81       * This would ensure it's changed exactly only once and not on every call
82       * to {@link #run}.
83       * 
84       * <h3>Examples</h3>
85       * <p>
86       * As the most simple use case, a client application might want to filter
87       * the initial setup for the archive file extensions it wants to detect.
88       * This is done as follows:
89       * <pre>{@code
90       * TConfig.get().setArchiveDetector(new TArchiveDetector("ear|jar|war"));
91       * }</pre>
92       * <p>
93       * This will filter all file system drivers found on the classpath in the
94       * initial setup so that only files with the pattern {@code *.ear},
95       * {@code *.jar} or {@code *.war} (case ignoring) are detected
96       * as prospective archive files.
97       * If no file system driver is present for a named extension in the initial
98       * setup, an {@link IllegalArgumentException} is thrown.
99       * <p>
100      * The constructors of the {@link TFile} class use the
101      * {@link TArchiveDetector} class to scan a path name for extensions of
102      * archive files which shall be treated like virtual directories.
103      * You can either explicitly inject this ArchiveDetector dependency into a
104      * TFile constructor or you can rely on the value of the class property
105      * {@code defaultArchiveDetector}.
106      * <p>
107      * By default, all new {@link TFile} objects will use a default
108      * {@link TArchiveDetector} which recognizes the canonical archive file
109      * extensions registered by all archive driver modules which are present on
110      * the runtime classpath.
111      * <p>
112      * However, if you use the following statement, all {@link TFile} objects
113      * will use the given default {@link TArchiveDetector} which recognizes
114      * only the given canonical file extensions for TAR, TAR.GZ, TAR.BZ2 and
115      * ZIP files as archive files and hence as virtual directories.
116      * This requires the JARs for the archive driver modules
117      * {@code truevfs-driver-tar} and {@code truevfs-driver-zip} to be present
118      * on the compile-time classpath:
119      * <pre>{@code
120      * TConfig.get().setArchiveDetector(
121      *         new TArchiveDetector(
122      *             TArchiveDetector.NULL,
123      *             new Object[][] {
124      *                 { "tar", new TarDriver(IOPoolLocator.SINGLETON) },
125      *                 { "tgz|tar.gz", new TarGZipDriver(IOPoolLocator.SINGLETON) },
126      *                 { "tbz|tb2|tar.bz2", new TarBZip2Driver(IOPoolLocator.SINGLETON) },
127      *                 { "zip", new ZipDriver(IOPoolLocator.SINGLETON)},
128      *             }));
129      * }</pre>
130      * <p>
131      * Another typical use case is to recognize only Java artifacts.
132      * <pre>{@code
133      * TConfig.get().setArchiveDetector(
134      *         new TArchiveDetector(
135      *             "ear|jar|war",
136      *             new JarDriver(IOPoolLocator.SINGLETON)));
137      * }</pre>
138      * <p>
139      * ... or an application file format.
140      * <pre>{@code
141      * TConfig.get().setArchiveDetector(
142      *         new TArchiveDetector(
143      *             "foo",
144      *             new JarDriver(IOPoolLocator.SINGLETON)));
145      * }</pre>
146      * <p>
147      * ... or an encrypted application file format.
148      * This driver authenticates input archive files up to 512 KB using the
149      * Message Authentication Code (MAC) specified by the RAES file format.
150      * For larger input archive files, it just checks the CRC-32 value whenever
151      * an archive entry input stream is closed.
152      * CRC-32 has frequent collisions when compared to a MAC.
153      * However, it should not be feasible to make an undetectable modification.
154      * The driver also uses unencrypted temporary files for archive entries
155      * whenever required.
156      * <pre>{@code
157      * TConfig.get().setArchiveDetector(
158      *         new TArchiveDetector(
159      *             "bar",
160      *             new SafeZipRaesDriver(IOPoolLocator.SINGLETON)));
161      * }</pre>
162      * <p>
163      * If you're a bit paranoid, then you could use the following driver
164      * instead:
165      * This driver authenticates every input archive file using the Message
166      * Authentication Code (MAC) specified by the RAES file format, which
167      * makes it comparably slow.
168      * The driver also uses unencrypted temporary files for archive entries
169      * whenever required.
170      * <pre>{@code
171      * TConfig.get().setArchiveDetector(
172      *         new TArchiveDetector(
173      *             "bar",
174      *             new ParanoidZipRaesDriver(IOPoolLocator.SINGLETON)));
175      * }</pre>
176      * <p>
177      * And finally, if you're quite paranoid, then this driver is for you:
178      * This driver authenticates every input archive file using the Message
179      * Authentication Code (MAC) specified by the RAES file format, which
180      * makes it comparably slow.
181      * The driver also uses unencrypted byte arrays for temporary storage of
182      * archive entries whenever required.
183      * If you were completely paranoid, you would even want to use encrypted
184      * byte arrays or wipe them with nulls after use.
185      * However, then you would have to write this yourself! ;-)
186      * <pre>{@code
187      * TConfig.get().setArchiveDetector(
188      *         new TArchiveDetector(
189      *             "bar",
190      *             new ParanoidZipRaesDriver(new ByteArrayIOPoolService(2048))));
191      * }</pre>
192      * <p>
193      * Last but not least, the following class property controls whether
194      * archive files and their member directories get automatically created
195      * whenever required.
196      * The default value of this class property is {@code true}!
197      * <pre>{@code
198      * TConfig.get().setLenient(false);
199      * }</pre>
200      */
201     @SuppressWarnings("NoopMethodInAbstractClass")
202     protected void setup() { }
203 
204     /**
205      * Runs the work phase.
206      * <p>
207      * This method is called by {@link #run} at least once and repeatedly
208      * called until it returns a non-negative integer for use as the
209      * {@link System#exit(int) exist status} of the VM.
210      * After this method, the {@link #sync} method is called in a finally-block.
211      * <p>
212      * Avoid repeating this method and updating the same archive file upon
213      * each call!
214      * This would degrade the overall performance from O(n) to O(m*n),
215      * where m is the number of new or modified entries and n is the number
216      * of all entries in the archive file!
217      * 
218      * @param  args an array of arguments for this application.
219      * @return A negative integer in order to continue calling this method
220      *         in a loop.
221      *         Otherwise, the return value is used as the
222      *         {@link System#exit(int) exit status} of the VM.
223      * @throws E At the discretion of the implementation.
224      */
225     protected abstract int work(String[] args) throws E;
226 
227     /**
228      * Runs the sync phase.
229      * <p>
230      * This method is called by {@link #run} in a finally-block after each call
231      * to {@link #work}.
232      * Its task is to commit any unsynchronized changes to the contents of
233      * archive files and purge all cached data so that third parties can safely
234      * interact with these archive files until the next access by this TrueVFS
235      * application.
236      * <p>
237      * The implementation in the class {@link TApplication} simply calls
238      * {@link TVFS#umount() TVFS.umount()}.
239      *
240      * @throws FsSyncWarningException if <em>only</em> warning conditions
241      *         occur.
242      *         This implies that the respective parent file system has been
243      *         updated with constraints, such as a failure to set the last
244      *         modification time of the entry for the federated file system
245      *         (i.e. archive file) in its parent file system.
246      * @throws FsSyncException if any error conditions occur.
247      *         This implies loss of data!
248      */
249     protected void sync() throws FsSyncException {
250         TVFS.umount();
251     }
252 }
